# PLOTARMOUR - MAIN SITE CONTEXT

## Overview

PlotArmour is a multi-vertical company building brands, products, games, merchandise systems, and culture-facing experiences.

This main website is not a services website and not a detailed portfolio.

Its job is to:

- establish the PlotArmour identity
- communicate scale, seriousness, and control
- present PlotArmour as a unified operating system
- route users into the correct vertical
- create trust through structure, tone, and selected work

Its job is not to:

- explain every service
- behave like an agency website
- overload users with long business copy
- duplicate what Studio or Store will later explain in depth

The site should make people feel:

"These people build serious things, and everything looks connected."

---

## Core Positioning

Primary line:

PlotArmour builds brands, products, games, and merchandise systems.

Current hero direction:

Built for what comes next.
From idea to execution, everything runs through a single system.

Strategic positioning:

PlotArmour operates as a unified system across creators, brands, communities, startups, products, media, fashion, and merchandise.
Execution is kept in-house across design, development, production, and distribution so the brand feels controlled, fast, and consistent.

Important:

- talk about systems, execution, structure, pipeline, and scale
- avoid sounding like a generic creative agency
- avoid sounding overly corporate or consultancy-like
- avoid listing long capability menus on the homepage

## Main Site Role

The main site is the parent layer above these ecosystems:

- Studio
- Merchandise
- Store

Recommended mental model:

- Main site = operating system / brand shell
- Studio = build and execution arm
- Merchandise = production and apparel systems arm
- Store = culture/fashion/consumer arm

The main site should feel broader and more architectural than the vertical sites.
It should frame the ecosystem, not replace the sub-sites.

## UX Principles

The experience should be optimized for clarity, confidence, and atmosphere.

Core UX principles:

- immediate brand impact in first viewport
- extremely clear path into each vertical
- low clutter and high legibility
- every section should have one job only
- interaction should feel smooth, calm, and premium
- strong hierarchy over excessive content

Homepage user flow:

1. Land and feel immediate impact
2. Understand PlotArmour as one system
3. See the three main verticals
4. Notice selected work as proof of range
5. Exit into Studio, Merchandise, Store, or Contact

The site must feel intentional, not crowded.
If a section starts behaving like a dashboard, agency page, or generic SaaS grid, it is going off-track.

## UI Theme

### General Theme

The main site should feel like a dark, engineered, premium interface.

Reference feeling:

- cinematic
- precise
- technical
- premium
- atmospheric
- slightly futuristic
- never flashy

This is not:

- playful startup UI
- clean white SaaS UI
- loud cyberpunk
- fashion-editorial maximalism

It should sit between:

- high-end product brand
- modern entertainment/IP company
- design-forward systems company

### Color Direction

Base:

- near-black background
- layered charcoal / deep graphite surfaces
- soft white typography

Accent behavior:

- very restrained orange, cyan, blue, violet, or muted spectral tones
- accents should feel embedded into atmosphere, not pasted on top
- gradients should be subtle, moody, and blurred

Recommended color logic:

- background: `#040507` / `#07090c`
- text: soft white, not pure harsh white everywhere
- borders: thin low-opacity white
- glow: very subtle orange/cyan accents

Do not use:

- bright flat purple as default accent
- candy gradients
- loud neon everywhere
- high-saturation brand blocks

### Typography

Typography is one of the main identity tools.

Desired feel:

- large, commanding, uppercase display type
- clean geometric or grotesk display face
- readable modern sans for body

Current direction:

- display: `Space Grotesk`
- body: `Sora`

Typography rules:

- brand name should dominate the hero
- section headlines should be short and forceful
- supporting copy should remain compact
- generous tracking for labels and utility text
- avoid paragraph-heavy layouts

### Spacing

Spacing should feel expensive.

Rules:

- large outer spacing
- roomy section gaps
- content breathes
- avoid dense stacking
- avoid tiny cards crammed into the viewport

## Motion System

Motion is essential to the site identity.
It should feel relaxed, weightless, and expensive.

Desired motion qualities:

- smooth
- relaxed
- soft
- gliding
- calm
- slightly cinematic

Avoid:

- snappy startup UI motion
- bouncy toy-like animation
- aggressive parallax everywhere
- chaotic hover effects

### Current Motion Pattern

- smooth scrolling across the app
- route transitions with blur/fade movement
- subtle hero particle movement
- animated sliding nav pill
- button lift and light sweep
- custom cursor with soft expansion on interactive targets

### Hero Motion

Hero contains a subtle particle field.

Particle rules:

- sparse dots
- low opacity
- slow floating motion
- slight cursor reaction
- no noisy swarm behavior
- only in hero or equivalent premium intro moments

### Scrolling

Scrolling should feel eased and relaxed, not hyper-fast.

Desired behavior:

- premium smooth-scroll feel
- stable and non-jittery
- no gimmicky overscroll tricks
- readable while moving

### Hover / Microinteraction Rules

Buttons:

- soft lift
- subtle scale
- light sweep or shimmer is acceptable
- no harsh bounce

Cards:

- slow surface shift or glow
- mild translate up
- gentle background depth movement

Navigation pill:

- should glide between active items
- should feel liquid and precise

Cursor:

- custom cursor is acceptable on desktop only
- it should feel thematic and subtle
- never obstruct reading
- disable automatically on touch devices or reduced-motion contexts

## Design Rules

### Always Do

- keep the interface dark, restrained, and premium
- prioritize hierarchy over content volume
- use bold display typography
- make sections feel intentional and distinct
- use motion to improve feel, not to show off
- preserve a system-level perspective
- make every major section scan quickly

### Never Do

- no long service lists on the homepage
- no generic agency layout
- no dashboard-card overload
- no stock-photo-driven design
- no loud neon cyberpunk
- no random illustrations
- no emoji
- no cluttered header
- no weak first impression

---

## Theme Rules For Future Vertical Sites

The Studio and Store websites should inherit the PlotArmour design DNA, but not copy the main site blindly.

What must stay consistent across all verticals:

- dark premium base
- strong typography
- smooth motion language
- controlled gradients and glows
- rounded premium surfaces
- sophisticated hover behavior
- high contrast and clean hierarchy
- same overall sense of engineering and taste

What should change per vertical:

### Studio

Studio should feel:

- sharper
- more technical
- more product-led
- more execution-oriented
- slightly more structured and systems-heavy

Studio accents can lean toward:

- cyan
- steel blue
- restrained orange

Studio content can go deeper into:

- brand building
- product design
- development
- game systems
- execution model

### Store

Store should feel:

- more cultural
- more fashion-led
- more editorial
- more image/product-forward
- more emotionally styled than the main site

Store accents can lean toward:

- richer tonal contrasts
- fashion-oriented gradients
- more expressive surface styling

Store content can go deeper into:

- drops
- labels
- products
- collections
- campaigns

Important:

The main site is the most architectural layer.
Studio should feel more operational.
Store should feel more cultural and product-led.

## Technical Direction

Recommended stack:

- Next.js App Router
- Tailwind CSS
- Framer Motion
- Lenis
- Three.js / react-three-fiber for hero particles
- Vercel for deployment

Current main-site interaction system includes:

- smooth app-wide scrolling
- route transitions
- animated nav pill
- animated CTA buttons
- desktop-only custom cursor
- hero particle field

Performance rules:

- motion should remain smooth on mid-range devices
- particles must stay subtle and lightweight
- degrade gracefully for touch and reduced-motion users
- do not let the custom cursor or animation hurt readability

The PlotArmour main site should feel like:
